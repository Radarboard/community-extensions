# Widget Naming And Capability Taxonomy

Use names that describe the user-visible data or workflow, not implementation details.

## Naming

- Good: `github-stars`, `npm-downloads`, `service-monitor`
- Avoid: `api-widget`, `new-dashboard-card`, `provider-data`

## Common capabilities

- `analytics`
- `app-reviews`
- `downloads`
- `errors`
- `revenue`
- `seo`
- `shipping`
- `sponsorship`
- `stars`
- `uptime`

## Ownership

Use `canonical` when the widget is the primary shared surface for a capability. Use `specialized` when it is intentionally narrower or provider-specific.
